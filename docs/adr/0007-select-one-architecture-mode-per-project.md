# Select one architecture mode per project

OnionCry will not run Clean Architecture and Vertical Slice architecture-specific validation at the same time. A project selects one Project Architecture Mode with `architecture.mode` in its configuration, using `cleanArchitecture` or `verticalSlice`; if no mode is configured, OnionCry defaults to `cleanArchitecture`. Mode-specific structural options live under the selected mode, such as `architecture.cleanArchitecture`, while rule severities remain under `rules`. Enabling a rule from another architecture-specific rule family is a configuration error rather than a silently ignored rule. This keeps architecture-specific rule families coherent: `cleanarch/*` rules protect layered dependency direction, while future `verticalslice/*` rules protect slice cohesion and cross-slice boundaries.
